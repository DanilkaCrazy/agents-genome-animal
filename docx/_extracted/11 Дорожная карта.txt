План развития системы GenomeAI AgroAnimals
Roadmap продукта и платформы (СНГ, молочный КРС, продуктивность)
Фокус: ML обязателен + AI-отчёты, on-prem по умолчанию
Версия: 1.0    Дата: 17.12.2025
1. Цели roadmap и принципы
Цель roadmap — довести MVP до повторяемого коммерческого продукта на фермах СНГ, сохраняя локальный контур и доказуемые ML-результаты.
Roadmap предполагает итерационную разработку: короткие циклы пилота → фиксация метрик → расширение данных/функций.
Ключевой продуктовый принцип: система дает решения (decision-support), а не «просто аналитику».
Ключевой технический принцип: любой текст AI-отчёта строится только на проверенном fact-pack (анти-галлюцинации).
Примечание: Этот документ дополняет ТЗ и Functional Requirements; здесь — последовательность и приоритеты, а не спецификация интерфейсов.

3. Дорожная карта по кварталам (12 месяцев)
Q1: Стабилизация MVP и пилот №1 (S0)
Данные/контракт: финализировать P0 data contract (lactations) + маппинги типовых выгрузок (СЕЛЭКС/1С/Excel).
QC: строгий QC, отчет об ошибках и приоритизация исправлений (UC-3).
ML: базовая модель (регрессия milk_305_kg) + time-split метрики (MAE/RMSE/Spearman).
Скоринг: индекс/остатки, ранжирование, флаги доверия (confidence).
AI: шаблон Executive + Full, валидатор fact-pack, единые дисклеймеры.
Процесс пилота: цикл «выгрузка→отчет→решения→обратная связь», фиксация причин принятия/отказа рекомендаций.
Результат Q1
1 ферма: минимум 2–3 прогона с улучшением качества данных.
Подтвержден формат отчетов и список действий, которые реально принимают.
Q2: Тиражирование на 2–3 фермы и стандарт внедрения (S1)
Онбординг: чек-лист внедрения (данные/юридика/ИБ), шаблоны NDA + приложения к договору по данным.
Стандартизация: конфиги маппинга, шаблоны входных файлов, единый run registry и структура артефактов.
Сравнимость: единые contemporary groups и политика качества групп, бенчмарки внутри фермы/площадок.
UX-упаковка: максимально простой сценарий запуска и упаковка результатов (файлы + папка прогона).
Контроль качества модели: отчет зон слабости (по parity/площадкам/периодам), политика «когда нельзя рекомендовать».
Результат Q2
3 фермы: повторяемая поставка отчётов; time-to-onboard сокращен; снижена ручная поддержка.
Сформирован коммерческий пакет MVP (что входит/не входит, условия пилота, прайс-логика).
Q3: Расширение данных P1 и рост ценности (S2)
P1 признаки: birth_date → age_at_calving (критичная фича).
ECM/FPCM: расчет и вывод отдельной вкладкой/секцией отчета (без замены P0 метрики).
SCC: флаги качества/риска (пока как флаги/колонки, без сложной оптимизации).
Test-day: при наличии — агрегирование в фичи по лактации (пики, темпы, DIM).
Объяснимость: стандартизировать интерпретацию (вклад признаков/сравнение с группой) для доверия.
Результат Q3
Рост Spearman/стабильности ранжирования и доверия к рекомендациям.
Расширенный AI-отчёт: больше контекста, но только из fact-pack.
Q4: Переход к платформе и интеграциям (S3)
Интеграции: коннекторы/регламенты выгрузок к учеткам и доильным системам; снижение ручного маппинга.
Мониторинг: контроль дрейфа данных и качества групп; алерты «данные испортились».
Multi-farm: каталог ферм/площадок, сравнение площадок, агрегированные KPI.
Безопасность: политики хранения/доступа; аудит доступа к отчетам.
Результат Q4
Сокращение времени получения отчета, рост надежности, готовность к масштабированию на холдинг.
4. Бэклог эпиков по направлениям
4.1 Продукт и отчеты
Ролевые версии отчетов: директор (1 стр.), зоотехник (полный), консультант (методология/QC).
Регистр решений и обратной связи: какие рекомендации приняты, почему.
Режимы доверия: при низком качестве данных/модели перевод рекомендаций в «наблюдение».
4.2 Данные и качество
Контроль изменений схемы (schema drift) и совместимость с разными выгрузками.
Библиотека типовых ошибок учетных систем + безопасные авто-правки (где допустимо).
Справочники: площадки, статусы, корректные диапазоны показателей.
4.3 ML и аналитика
Базовые модели: линейные и GBDT; критерий выбора — Spearman/стабильность + объяснимость.
Калибровка confidence: QC + size группы + интервал неопределенности + качество модели.
Дальше: иерархические модели для холдингов, multi-task (удой + качество).
4.4 LLM (AI-отчет)
Fact-pack + валидатор чисел как обязательный контур безопасности.
Словарь терминов и «разрешенные» формулировки (дисклеймеры, запреты, стиль).
Локальные LLM/endpoint для on-prem; настройка под отраслевую лексику.
4.5 Экосистема и продажи
Пакет внедрения: чек-лист, шаблоны договоров, требования ИБ.
Партнерства: интеграторы учетных систем, ассоциации, консалтинг.
Каталог открытых датасетов для демо + синтетический демо-набор (без коммерческой тайны).
