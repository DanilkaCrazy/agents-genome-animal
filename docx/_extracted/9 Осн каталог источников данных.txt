GenomeAI — Каталог открытых табличных источников данных (дополнение)
Версия: v1.1
Дата: 17.12.2025
Цель этого дополнения — расширить перечень открытых источников с табличными данными, которые можно использовать для обучения ML, тестовых показов и отладки пайплайнов (ингест → QC → фичи → обучение → AI-отчёт).
Критерии “подходит под условия документации”
желательно наличие `animal_id` / `cow_id`
желательно наличие `parity` / `lactation_number`
`calving_date` или (`record_date` + `days_in_milk`)
`milk_305_kg` или test-day/суточные удои для расчёта 305d
Табличные открытые датасеты, наиболее совместимые с MVP (P0/P1)
Примечания по юридике и комплаенсу
Всегда перепроверяйте лицензию в карточке набора (особенно Kaggle/GitHub): коммерческое использование, атрибуция, ограничения на AI training / text & data mining.
Даже при открытой лицензии держите происхождение данных и ссылку на источник в реестре датасетов проекта (например, data_catalog.yml / README), чтобы корректно оформить атрибуцию.
Для тестовых демонстраций без рисков используйте синтетические наборы; реальные — только в R&D контуре.
Рекомендация по приоритизации
Zenodo Dummy milking robot — для витрин, отчётов, пайплайна, тестов.
Mendeley Czech parity/breed — быстро собрать P0 модель по лактациям + contemporary groups.
Dryad mountain-pastured — добавить пример внешних факторов (THI) и кейс QC по датам/лактациям.
Mendeley pedigree+performance — для демонстрации P1 “mating plan по родословной”.
