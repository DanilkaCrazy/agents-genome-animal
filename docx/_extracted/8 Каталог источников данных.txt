GenomeAI AgroAnimals — Каталог открытых источников данных для ML и демо (молочный КРС)
Версия: v1.0 | Дата: 17.12.2025 | Регион фокуса продукта: СНГ (источники данных: глобальные, т.к. открытых cow-level по СНГ почти нет)
1. Назначение документа
Документ фиксирует перечень публично доступных (open) и условно открытых (semi-open) источников данных, которые можно использовать для: (a) обучения/прототипирования ML-моделей продуктивности, (b) тестовых показов (demo) и отладки пайплайнов, (c) построения бенчмарков и контекстных метрик.
2. Важные ограничения
1) В СНГ данные уровня «животное–лактация/тест-день» обычно являются коммерческой тайной и ведутся в СЕЛЭКС/1С/ПО доильного зала; публичные датасеты такого уровня встречаются крайне редко.
2) Для MVP-качества, пригодного к принятию решений на ферме, почти наверняка потребуется обучение на данных пилотных ферм (по NDA/договору). Открытые датасеты — в основном для демонстрации и ускорения разработки.
3) Лицензии/условия доступа нужно проверять перед загрузкой в репозиторий и коммерческим использованием (особенно Kaggle и «публичные каталоги быков»).
3. Какие поля нужны системе (привязка к P0/P1)
P0 (минимум для модели продуктивности): animal_id, parity, calving_date, milk_305_kg (или test-day, чтобы посчитать 305), желательно farm_id/группа сравнения.
P1 (расширение): birth_date (age_at_calving), sire_id/dam_id (родословная), SCC/здоровье, события воспроизводства (AI/отёл/выбраковка), рационы/группы содержания, погодные факторы.

5. Рекомендации по использованию для MVP
Рекомендуемый минимальный набор для демо/отладки пайплайна:
• Mendeley (Czech 659 cows) — основной табличный набор для P0 рейтинга.
• Mendeley (Sires + pedigree) — для демонстрации P1 mating plan.
• Mendeley (SCC test-day) — для флагов здоровья и объяснимости отчёта.
• Zenodo (Taenikon) + Weather (NASA POWER/ERA5) — если хотите показать влияние погоды/THI.
6. Что делать, если нужны именно данные СНГ на уровне животных
Для СНГ-пилота обычно используется путь: (1) подписать NDA и DPA/договор оказания услуг с фермой/холдингом; (2) выгрузить минимальный пакет P0 из СЕЛЭКС/1С/ПО доильного зала (XLSX/CSV); (3) провести первичный QC и согласовать словарь полей; (4) обучить и валидировать модель на данных конкретной фермы; (5) выпускать AI-отчёт регулярно.
